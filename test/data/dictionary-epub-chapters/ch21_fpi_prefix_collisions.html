21. Optimized Index (.fpi) -- Prefix Collisions

These tests exercise the .idx.fpi optimized index (fencep_6 + tab1_3): the exact-match fast lookup path it accelerates, and prefix-collision correctness.

Test A: Baseline lookup on a dictionary with optimized index

The english-full dictionary ships with english-full.idx.fpi. Every lookup must return the correct definition. This implicitly exercises the optimized-index fast path because the device tries it first when present.

Dictionary

english-full

Words

  • apple
  • bridge
  • cloud
  • dawn
  • echo

Steps

  1. Select english-full in the picker. No preparation should be required.
  2. Look up each word above.
  3. Each returns a valid definition. No errors. No crash.

Test B: Prefix-collision disambiguation

The prep-fpi-prefix-collision dictionary contains headwords that share 16-character prefixes. Each word must still return its own distinct definition -- not a sibling's. Requires Ch 4 Test F to have been run so this dictionary has been prepared.

Dictionary

prep-fpi-prefix-collision

Words

16-char prefix international:

  • internationalism
  • internationalize
  • internationalized

16-char prefix microelectromech:

  • microelectromechanical
  • microelectromechanics

Control words (no collision):

  • antidisestablish
  • simple
  • test

Steps

  1. Select prep-fpi-prefix-collision in the picker. If prep steps are listed, Ch 4 Test F was skipped -- run that first, then return here.
  2. Look up each word above.
  3. Each word returns its own distinct definition. Compare against the definition text in test/data/dictionary-sources/prep-fpi-prefix-collision.json if in doubt.
  4. No two colliding words return the same definition.
